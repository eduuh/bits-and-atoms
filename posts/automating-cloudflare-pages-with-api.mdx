I needed staging and production domains for my blog on Cloudflare Pages. You can do this in the dashboard, but I just didn't feel like clicking around. The API was right there.

## What I wanted

Two domains, one project:

- `eduuh.com` → `blog-2026` main branch
- `staging.eduuh.com` → `blog-2026` staging branch

## Getting API access

Grabbed an API token from [the Cloudflare dashboard](https://dash.cloudflare.com/profile/api-tokens). The scopes you need are `Account > Cloudflare Pages > Edit`, `Account > Account Settings > Read`, and (if you also want DNS) `Zone > DNS > Edit`.

<Callout type="warning" title="Don't be that guy">
  Keep the token out of git. I stuck mine in `.env` and added it to `.gitignore`.
</Callout>

## Adding the domains

Attaching custom domains to a Pages project:

```bash
# Root domain
curl -X POST \
  "https://api.cloudflare.com/client/v4/accounts/{account_id}/pages/projects/blog-2026/domains" \
  -H "Authorization: Bearer {token}" \
  -H "Content-Type: application/json" \
  --data '{"name":"eduuh.com"}'

# Staging subdomain
curl -X POST \
  "https://api.cloudflare.com/client/v4/accounts/{account_id}/pages/projects/blog-2026/domains" \
  -H "Authorization: Bearer {token}" \
  -H "Content-Type: application/json" \
  --data '{"name":"staging.eduuh.com"}'
```

## Pointing staging at the staging branch

Here's where I got stuck. Both domains were now attached, but both were serving `main`. Not what you want from a staging URL.

You have to tell Cloudflare which branch each domain should serve:

```bash
curl -X PATCH \
  "https://api.cloudflare.com/client/v4/accounts/{account_id}/pages/projects/blog-2026/domains/staging.eduuh.com" \
  -H "Authorization: Bearer {token}" \
  -H "Content-Type: application/json" \
  --data '{"branch":"staging"}'
```

That was the bit I couldn't figure out in the UI. The API just says it.

## DNS while I'm here

The domain's already on Cloudflare, so I set up DNS through the API too:

```bash
# Root domain
curl -X POST \
  "https://api.cloudflare.com/client/v4/zones/{zone_id}/dns_records" \
  -H "Authorization: Bearer {token}" \
  -H "Content-Type: application/json" \
  --data '{
    "type": "CNAME",
    "name": "@",
    "content": "blog-2026-290.pages.dev",
    "proxied": true
  }'

# Staging
curl -X POST \
  "https://api.cloudflare.com/client/v4/zones/{zone_id}/dns_records" \
  -H "Authorization: Bearer {token}" \
  -H "Content-Type: application/json" \
  --data '{
    "type": "CNAME",
    "name": "staging",
    "content": "blog-2026-290.pages.dev",
    "proxied": true
  }'
```

Both records point at the same `pages.dev` subdomain. Cloudflare picks the right branch based on the domain config from earlier.

## That's it

| Domain | Branch |
|--------|--------|
| `eduuh.com` | main |
| `staging.eduuh.com` | staging |

Push to `main`, production updates. Push to `staging`, staging updates.

The `PATCH` to pin a branch to a domain was the bit I couldn't find in the dashboard. Maybe it's there somewhere. The API was clearer.
